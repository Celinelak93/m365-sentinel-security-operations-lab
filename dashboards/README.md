# Dashboards and Reports

This folder contains management-facing reporting examples for the security operations lab.

The purpose is to show how technical security findings can be translated into executive summaries, operational metrics, and risk-based recommendations.

## Current Reports

| Report | Purpose |
|---|---|
| `executive-incident-report.md` | Summarizes a phishing URL click incident for management |

## Reporting Goals

Security reporting should answer:

1. What happened?
2. Who was affected?
3. Was compromise confirmed?
4. What business risk exists?
5. What actions were taken?
6. What should be improved next?
