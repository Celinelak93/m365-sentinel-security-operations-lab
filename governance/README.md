# Governance

This folder contains governance and compliance-oriented materials for the security operations lab.

The goal is to show how technical security operations connect to risk management, incident handling, executive reporting, and NIS2-style readiness.

## Files

| File | Purpose |
|---|---|
| `security-control-inventory.md` | Maps asset areas to risks, controls, detections, gaps, and improvements |
| `nis2-readiness-mapping.md` | Maps project controls to NIS2-style governance expectations |
| `incident-notification-decision-workflow.md` | Provides a decision workflow for incident escalation and notification review |

## Governance Goals

This folder helps answer:

1. What assets and risks are covered?
2. What controls exist?
3. What gaps remain?
4. What evidence can be produced after an incident?
5. How are incidents escalated?
6. How are security improvements prioritized?
