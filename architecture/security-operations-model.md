# Security Operations Model

## Purpose

This document explains how a Microsoft-based security operations capability can be designed for a small or medium-sized organization.

The model connects assets, security tools, detections, incident response, governance, and management reporting.

## Security Operations Flow

The security operations process follows this flow:

Assets
  ↓
Security Controls
  ↓
Log Sources
  ↓
Microsoft Defender XDR / Microsoft Sentinel
  ↓
Detections and Analytics Rules
  ↓
Incidents
  ↓
Triage and Investigation
  ↓
Containment and Remediation
  ↓
Reporting and Governance
